# 代码统计工具需求规格

## 1. 概述

基于 `./MinClaudeCode/count_lines.py` 的现有实现进行增强，开发一个 Python 代码行数统计工具 `code_statistics.py`，存放在 `./MinClaudeCode/` 目录下。

## 2. 现有功能总结（来源于 count_lines.py）

`count_lines.py` 已实现以下能力：

| 功能项 | 说明 |
|--------|------|
| **目录遍历** | 使用 `pathlib.Path.rglob("*")` 递归扫描项目目录 |
| **跳过规则** | 跳过 `.git`、`.idea`、`__pycache__`、`venv`、`.venv`、`node_modules`、`code_output`、`log`、`context`、`.memdir` 等目录；跳过 `.gitignore`、`count_lines.py` 等文件；跳过隐藏文件（以 `.` 开头） |
| **扩展名过滤** | 仅统计 `.py`、`.yaml`、`.yml`、`.json`、`.md`、`.txt`、`.sh`、`.bat`、`.ps1`、`.toml`、`.ini`、`.cfg` |
| **行分类统计** | 对每个扩展名分类统计：总行数、代码行、空行、注释行。Python 注释识别 `#`、`"""`、`'''`；YAML/JSON/INI/CFG/TOML 注释识别 `#`、`//` |
| **汇总输出** | 使用 Rich Table 渲染按扩展名分组的统计表格，包含文件数、总行数、代码行、空行、注释行，最后输出总计行 |
| **人天估算** | 按人均 300~500 行有效代码/天给出估算参考 |

核心逻辑：`总行数 = 代码行 + 空行 + 注释行`，其中 `代码行 = 总行数 - 空行 - 注释行`。

## 3. 新增需求：详细统计

在现有按扩展名汇总统计的基础上，增加**详细统计**功能：

### 3.1 各目录下各文件的代码行数统计

- 对项目中的每个**目录**，分别列出该目录下所有被统计的文件。
- 每个文件显示：**文件名**、**总行数**、**代码行**、**空行**、**注释行**。
- 每个目录显示该目录的汇总行（文件数、总行数、代码行、空行、注释行）。
- 目录的层级关系需要在输出中清晰体现（如按目录分组展示）。

### 3.2 表格展示

- 详细统计结果必须以 **Rich Table** 渲染的表格形式展示。
- 每个目录一张表格，表头包含：文件名、总行数、代码行、空行、注释行。
- 表格末尾附带该目录的汇总行。
- 所有目录统计完毕后，最后输出一张**项目总计表**（与现有 `count_lines.py` 的汇总表格式一致，按扩展名分组 + 总计）。

### 3.3 输出顺序

1. 先逐目录输出详细统计表（每个目录一张表格）
2. 最后输出项目总计汇总表

## 4. 技术约束

| 约束项 | 要求 |
|--------|------|
| **语言** | Python 3.12 |
| **依赖** | `pathlib`、`rich`（Rich Table），不引入额外第三方库 |
| **文件名** | `code_statistics.py` |
| **存放路径** | `./MinClaudeCode/code_statistics.py`（项目根目录） |
| **跳过规则** | 沿用 `count_lines.py` 的跳过目录列表和跳过文件列表，额外跳过 `code_statistics.py` 自身 |
| **扩展名** | 沿用 `count_lines.py` 的扩展名列表 |
| **注释识别** | 沿用 `count_lines.py` 的注释识别逻辑 |
| **编码** | UTF-8，`errors='ignore'` |

## 5. 特别说明

- **统计包含自身**：`code_statistics.py` 的代码行数**必须被纳入统计范围**（即它统计项目时，也把自己算进去）。`count_lines.py` 原逻辑跳过自身（`skip_files = {'.gitignore', 'count_lines.py'}`），新实现中 `skip_files` 不应包含 `code_statistics.py`。

## 6. 测试要求

- 生成代码后，**必须使用 `add_tests` skill** 为 `code_statistics.py` 编写 pytest 单元测试。
- 测试文件命名为 `test_code_statistics.py`，存放于 `./MinClaudeCode/tests/` 目录下（如目录不存在则创建）。
- 测试覆盖要点：
  - 跳过目录规则正确
  - 跳过文件规则正确（验证 `code_statistics.py` 自身被统计、`count_lines.py` 被跳过）
  - 扩展名过滤正确
  - 行分类（代码/空行/注释）统计准确
  - 目录级别汇总正确
  - 项目总计正确
- **所有测试用例通过后，代码统计任务才算完成。**
